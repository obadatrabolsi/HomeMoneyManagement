# الديون والقروض — مواصفة تصميم (المرحلة الثانية، الجزء 4)

> التاريخ: 2026-06-29
> النطاق: الديون والقروض فقط. يبني على الـ MVP + الميزانيات + الأهداف + العمليات المتكررة المدموجين في `master`.

## 1. نظرة عامة

إدارة الأموال التي على المستخدم دفعها («عليّ») والأموال التي يجب أن يستلمها («لي»)، مع سجل دفعات لكل دين، ومبلغ مدفوع/متبقٍ ونسبة سداد. عربية RTL، دون اتصال، المبالغ بالسنتات.

## 2. القرارات المعتمدة

| القرار | الاختيار |
| --- | --- |
| الاتجاه | `owe` (عليّ، أدفع) و `owed` (لي، أستلم) |
| التقدّم | المدفوع = مجموع **دفعات** مسجّلة (جدول مستقل)؛ المتبقي محسوب |
| أثر الدفعة على الأرصدة | **لا أثر** على أرصدة الحسابات هذه المرحلة (تتبّع فقط؛ الربط بعملية مالية تحسين لاحق) |
| العملة | لكل دين عملة واحدة (لا تحويل)؛ الدفعات بنفس عملة الدين |
| التسوية | `isSettled` يدوي + اعتبار الدين مسوّى تلقائيًا للعرض عند بلوغ المدفوع المبلغ |
| الحذف | حذف فعلي للدين (مع دفعاته) |
| ترقية المخطط | Dexie v5 — جدولان جديدان `debts` و `debtPayments` (إضافي فقط) |
| التوافق العكسي | استيراد JSON يقبل `schemaVersion` 1..5؛ الجداول الغائبة `[]` |
| الوصول | صفحة `/debts` + رابط في صفحة الإعدادات |

## 3. نموذج البيانات

### جدول `debts`
`id` · `direction` (`owe`|`owed`) · `person` · `amount` (سنتات، الإجمالي) · `currency` · `dueDate?` (`yyyy-MM-dd`) · `notes?` · `isSettled` (bool) · `createdAt` · `updatedAt`

### جدول `debtPayments`
`id` · `debtId` · `amount` (سنتات) · `date` (`yyyy-MM-dd`) · `note?` · `createdAt`

### ترقية `SCHEMA_VERSION` إلى 5
`db.version(5).stores({ ...الجداول الحالية, debts: 'id, direction, isSettled', debtPayments: 'id, debtId' })`. تبقى الإصدارات 1–4 كما هي.

## 4. المستودع `debtsRepo`

- `createDebt(input): Promise<Debt>` — `input = { direction, person, amount, currency, dueDate?, notes? }`؛ `isSettled=false`.
- `updateDebt(id, patch)` · `deleteDebt(id)` (يحذف الدين ودفعاته) · `listDebts(filter?: { direction?; includeSettled? }): Promise<Debt[]>`.
- `addPayment(input): Promise<DebtPayment>` — `input = { debtId, amount, date, note? }`.
- `listPayments(debtId): Promise<DebtPayment[]>` (الأحدث أولًا).
- `DebtProgress = { debt; paid; remaining; percent; settled }` (`settled = isSettled || (amount>0 && paid>=amount)`؛ `percent = amount>0 ? round(paid/amount*100) : 0`؛ `remaining = max(amount-paid,0)`).
- `debtsWithProgress(filter?): Promise<DebtProgress[]>`.
- `remainingTotalsByDirection(): Promise<{ owe: Record<string,number>; owed: Record<string,number> }>` — مجموع المتبقي لكل اتجاه مجمّعًا حسب العملة (للملخص).

## 5. النسخ الاحتياطي

- `exportBackup` يضيف `debts` و `debtPayments` (الإصدار 5).
- `importBackup` يقبل `schemaVersion ∈ {1..5}`؛ غياب الجدول → `[]`.

## 6. الواجهة

- **صفحة `/debts`:** قسمان «عليّ» (owe) و«لي» (owed)؛ كل دين: اسم الشخص، شريط تقدّم، «المدفوع / الإجمالي»، المتبقي، تاريخ الاستحقاق إن وُجد، زر «إضافة دفعة»، وحذف. وسم «مسوّى ✓» عند اكتمال السداد. ملخص أعلى الصفحة: إجمالي ما عليّ وما لي (حسب العملة).
- **`DebtForm`:** مبدّل اتجاه (عليّ/لي)، اسم الشخص، المبلغ، العملة (افتراضي من الإعدادات)، تاريخ استحقاق اختياري، ملاحظات.
- **`PaymentForm`:** مبلغ، تاريخ (افتراضي اليوم)، ملاحظة اختيارية.
- **صفحة الإعدادات:** إضافة رابط «الديون والقروض» إلى كتلة الروابط.

## 7. الاختبار

- **وحدة:** ترقية المخطط v5؛ `debtsRepo` (CRUD، حذف الدين يحذف دفعاته، دفعات وتقدّم وتسوية، `amount=0`، تصفية اتجاه/مسوّى، `remainingTotalsByDirection` لا يجمع عملات مختلفة)؛ النسخ الاحتياطي (تضمين الجدولين، استيراد v1–v5).
- **مكونات:** عرض `DebtsPage` لقسمي «عليّ/لي».

## 8. معايير القبول

1. ترقية المخطط إلى v5 دون فقدان بيانات v1–v4.
2. إنشاء/تعديل/حذف دين في أيٍّ من الاتجاهين، وإضافة دفعات تُحدّث المدفوع/المتبقي/النسبة.
3. وسم «مسوّى» عند اكتمال السداد.
4. ملخص «عليّ/لي» مجمّع حسب العملة (دون جمع عملات مختلفة).
5. النسخ الاحتياطي يضمّن الديون والدفعات، والاستيراد يقبل v1–v5.
