# الأهداف المالية — مواصفة تصميم (المرحلة الثانية، الجزء 2)

> التاريخ: 2026-06-29
> النطاق: ميزة الأهداف المالية فقط. يبني على الـ MVP + الميزانيات المدموجين في `master`.

## 1. نظرة عامة

أهداف ادخار (صندوق طوارئ، سفر، شراء جهاز…) لكل منها مبلغ مستهدف وتاريخ إنجاز اختياري ونسبة تقدّم. المستخدم يضيف «مساهمات» تدفع التقدّم نحو الهدف. عربية RTL، تعمل دون اتصال، المبالغ بالسنتات.

## 2. القرارات المعتمدة

| القرار | الاختيار |
| --- | --- |
| نموذج التقدّم | المبلغ الحالي = مجموع **مساهمات** مسجّلة (جدول مستقل)، وليس حقلًا مخزّنًا |
| أثر المساهمة على الأرصدة | **لا أثر** على أرصدة الحسابات في هذه المرحلة (متعقّب ادخار مستقل؛ الربط بتحويل من حساب تحسين لاحق) |
| العملة | لكل هدف عملة واحدة (لا تحويل)؛ المساهمات بنفس عملة الهدف |
| المساهمة السالبة | مسموحة (سحب/تصحيح) |
| الأرشفة | أرشفة الهدف (حذف ناعم) كالحسابات؛ المساهمات تُحذف فعليًا مع الهدف عند الأرشفة؟ لا — تبقى مرتبطة، والهدف المؤرشف يختفي من القوائم |
| ترقية المخطط | Dexie v3 — جدولان جديدان `goals` و `goalContributions` (إضافي فقط) |
| التوافق العكسي | استيراد JSON يقبل `schemaVersion` 1 و 2 و 3؛ الجداول الغائبة تُعامَل كقوائم فارغة |
| الوصول من الواجهة | قسم «الأهداف» في لوحة التحكم + روابط في صفحة الإعدادات (دون تبويب سفلي سادس)؛ وإصلاح وصول «التصنيفات» المعزول بإضافة رابط له أيضًا |

## 3. نموذج البيانات

### جدول `goals`
`id` · `name` · `targetAmount` (سنتات) · `currency` · `targetDate?` (`yyyy-MM-dd`) · `notes?` · `icon` · `color` · `isArchived` · `sortOrder` · `createdAt` · `updatedAt`

### جدول `goalContributions`
`id` · `goalId` · `amount` (سنتات، قد يكون سالبًا) · `date` (`yyyy-MM-dd`) · `note?` · `createdAt`

### ترقية `SCHEMA_VERSION` إلى 3
`db.version(3).stores({ ...الجداول الحالية, goals: 'id, isArchived, sortOrder', goalContributions: 'id, goalId' })`. تبقى `version(1)` و `version(2)` كما هما.

## 4. المستودع `goalsRepo`

- `createGoal(input): Promise<Goal>` — `input = { name, targetAmount, currency, targetDate?, notes? }` (icon/color/sortOrder/isArchived افتراضية).
- `updateGoal(id, patch): Promise<void>`
- `archiveGoal(id): Promise<void>`
- `listGoals(includeArchived?): Promise<Goal[]>` مرتّبة حسب `sortOrder`.
- `addContribution(input): Promise<GoalContribution>` — `input = { goalId, amount, date, note? }`.
- `listContributions(goalId): Promise<GoalContribution[]>`
- `GoalProgress = { goal: Goal; current: number; remaining: number; percent: number; reached: boolean }`
- `goalsWithProgress(includeArchived?): Promise<GoalProgress[]>` — `current` = مجموع مساهمات الهدف؛ `percent = target>0 ? round(current/target*100) : 0` (يُقصّ للعرض عند 100)؛ `reached = current >= target && target > 0`؛ `remaining = max(target - current, 0)` للعرض، مع إتاحة القيمة الخام عند الحاجة.

## 5. النسخ الاحتياطي

- `exportBackup` يضيف `goals` و `goalContributions` (الإصدار 3).
- `importBackup` يقبل `schemaVersion ∈ {1,2,3}`؛ الجداول الغائبة `[]`. يبقى الاستبدال الكامل.

## 6. الواجهة

- **صفحة الأهداف `/goals`:** قائمة الأهداف النشطة، كل عنصر: الاسم، شريط تقدّم، «الحالي / المستهدف»، النسبة، المتبقي، تاريخ الإنجاز إن وُجد، زر «إضافة مبلغ»، وأرشفة. عند بلوغ الهدف وسمٌ «تم الإنجاز ✓».
- **`GoalForm`:** اسم، مبلغ مستهدف، عملة (افتراضي من الإعدادات)، تاريخ إنجاز اختياري، ملاحظات.
- **`ContributionForm`:** مبلغ، تاريخ (افتراضي اليوم)، ملاحظة اختيارية.
- **لوحة التحكم:** قسم «الأهداف» يعرض حتى 3 أهداف بأشرطة تقدّم + رابط «عرض الكل» إلى `/goals` (يُعرض فقط عند وجود هدف).
- **صفحة الإعدادات:** كتلة روابط إلى «الأهداف» و«الميزانيات» و«التصنيفات» (تحسّن الوصول وتصلح عزل التصنيفات).

## 7. الاختبار

- **وحدة:** ترقية المخطط v3 واستمرار البيانات؛ `goalsRepo` (CRUD/أرشفة، مساهمات موجبة/سالبة، حساب التقدّم وبلوغ الهدف، target=0)؛ النسخ الاحتياطي (تضمين الجدولين، استيراد v1/v2/v3).
- **مكونات:** عرض `GoalsPage`، إضافة مساهمة تُحدّث التقدّم.

## 8. معايير القبول

1. ترقية المخطط إلى v3 دون فقدان بيانات v1/v2.
2. إنشاء/تعديل/أرشفة هدف، وإضافة مساهمات (موجبة وسالبة) تُحدّث المبلغ الحالي والنسبة.
3. وسم «تم الإنجاز» عند بلوغ المستهدف.
4. لوحة التحكم تعرض قسم الأهداف؛ صفحة الإعدادات توفّر روابط الوصول.
5. النسخ الاحتياطي يضمّن الأهداف والمساهمات، والاستيراد يقبل v1/v2/v3.
