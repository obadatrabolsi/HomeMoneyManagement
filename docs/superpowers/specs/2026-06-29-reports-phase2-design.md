# التقارير والإحصائيات — مواصفة تصميم (المرحلة الثانية، الجزء 5 / الأخير)

> التاريخ: 2026-06-29
> النطاق: التقارير والإحصائيات. يبني على كل ما سبق المدموج في `master`. **لا تغيير على المخطط أو النسخ الاحتياطي** (طبقة قراءة فقط).

## 1. نظرة عامة

تقارير ورسوم بيانية وإحصاءات محسوبة فوق العمليات والحسابات والتصنيفات الموجودة: مقارنة الدخل والمصروف، التوزيع حسب التصنيف، السلسلة الشهرية، ومجموعة إحصاءات. عربية RTL، دون اتصال، المبالغ بالسنتات.

## 2. القرارات المعتمدة

| القرار | الاختيار |
| --- | --- |
| تخزين | لا جداول جديدة ولا ترقية مخطط — حساب فوري فوق البيانات الحالية |
| العملة | كل التقارير **محصورة بعملة واحدة** يختارها المستخدم (افتراضي عملة الإعدادات)؛ تُحسب من عمليات الحسابات بتلك العملة فقط (لا تحويل) |
| الفترة | مُحدِّد: هذا الشهر / الشهر الماضي / هذه السنة / نطاق مخصّص (افتراضي هذا الشهر) |
| الرسوم | Recharts (موجود): مخطط أعمدة شهري (دخل/مصروف)، مخطط دائري للتصنيفات |
| الإحصاءات | أكبر مصروف، أكبر دخل، متوسط الإنفاق اليومي، أكثر تصنيف إنفاقًا، أكثر حساب استخدامًا، عدد العمليات |
| الوصول | صفحة `/reports` + رابط في صفحة الإعدادات |

## 3. المستودع `reportsRepo` (قراءة فقط)

دالة مساعدة داخلية: خريطة `accountId → currency`، تُستخدم لحصر العمليات بعملة التقرير. كل الدوال تستبعد العمليات المحذوفة وتستبعد التحويلات من حسابات الدخل/المصروف.

- `incomeExpenseTotals(from, to, currency): Promise<{ income: number; expense: number; net: number }>`
- `categorySpending(from, to, currency): Promise<Array<{ categoryId: string | null; total: number }>>` — مصروفات مجمّعة حسب التصنيف، تنازليًا.
- `monthlyTotals(year, currency): Promise<Array<{ month: string; income: number; expense: number }>>` — 12 عنصرًا (`yyyy-01`..`yyyy-12`).
- `statistics(from, to, currency): Promise<Statistics>` حيث:
  ```
  Statistics = {
    transactionCount: number
    largestExpense: Transaction | null
    largestIncome: Transaction | null
    avgDailyExpense: number          // إجمالي المصروف ÷ عدد أيام الفترة (سنتات)
    topCategoryId: string | null
    topCategoryTotal: number
    mostUsedAccountId: string | null // الحساب صاحب أكبر عدد عمليات ضمن العملة/الفترة
  }
  ```

## 4. الواجهة

- **صفحة `/reports`:**
  - **مُحدِّدا العملة والفترة** أعلى الصفحة.
  - **ملخّص الدخل/المصروف/الصافي** للفترة.
  - **مخطط أعمدة شهري** (دخل مقابل مصروف لكل شهر من سنة الفترة) — `MonthlyBar` بـ Recharts.
  - **مخطط دائري** لتوزيع المصروفات حسب التصنيف (إعادة استخدام نمط `CategoryPie`).
  - **بطاقات إحصائية:** أكبر مصروف/دخل، متوسط الإنفاق اليومي، أكثر تصنيف، أكثر حساب استخدامًا، عدد العمليات.
- **صفحة الإعدادات:** إضافة رابط «التقارير» إلى كتلة الروابط.

## 5. الاختبار

- **وحدة:** `reportsRepo` — حصر العملة، استبعاد المحذوف والتحويلات، `incomeExpenseTotals`/`categorySpending`/`monthlyTotals`/`statistics` (أكبر مصروف/دخل، متوسط يومي، أكثر تصنيف/حساب، عدد العمليات) على بيانات معدّة.
- **مكونات:** عرض `ReportsPage` (الملخّص والعناوين)؛ تجاهل قياس أبعاد Recharts في jsdom (لا توكيد على الرسم).

## 6. معايير القبول

1. كل الأرقام محصورة بعملة التقرير المختارة (لا تجميع عبر العملات).
2. ملخّص دخل/مصروف/صافٍ صحيح للفترة.
3. مخطط شهري ومخطط تصنيفات يعرضان بيانات الفترة/العملة.
4. الإحصاءات الست محسوبة بدقة.
5. الوصول من الإعدادات؛ لا تغيير على المخطط أو النسخ الاحتياطي.
