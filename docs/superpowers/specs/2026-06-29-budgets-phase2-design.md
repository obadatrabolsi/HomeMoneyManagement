# الميزانيات — مواصفة تصميم (المرحلة الثانية، الجزء 1)

> التاريخ: 2026-06-29
> النطاق: ميزة الميزانيات فقط. بقية المرحلة الثانية (الأهداف، المتكررة، الديون، التقارير) لها دورات مستقلة.
> يبني على الـ MVP (المرحلة الأولى) المدموج في `master`.

## 1. نظرة عامة

ميزانيات شهرية لكل تصنيف مصروفات، تعرض القيمة والمصروف والمتبقي ونسبة الاستهلاك، مع تنبيه بصري عند الاقتراب من التجاوز أو تجاوزه. تعمل بالكامل دون اتصال، عربية RTL، والمبالغ بالسنتات.

## 2. القرارات المعتمدة

| القرار | الاختيار |
| --- | --- |
| نطاق الميزانية | شهرية، لكل (تصنيف + شهر `yyyy-MM` + عملة) |
| العملة | لكل ميزانية عملة؛ المصروف يُحسب من حسابات بنفس العملة فقط (لا تحويل) |
| حساب المصروف | مجموع عمليات `expense` غير المحذوفة لذلك التصنيف، في ذلك الشهر، ضمن حسابات بعملة الميزانية |
| عتبة التنبيه | `near` عند ≥ 80%، `over` عند ≥ 100% |
| التنبيهات | بصرية فقط (لون + وسم)؛ إشعارات النظام مؤجَّلة للمرحلة 3 |
| ترقية المخطط | Dexie الإصدار 2 — إضافة جدول `budgets` (بدون أي تغيير على الجداول الحالية) |
| التوافق العكسي | استيراد JSON يقبل نسخ `schemaVersion` 1 و 2؛ الميزانيات الغائبة تُعامَل كقائمة فارغة |

## 3. نموذج البيانات

### جدول جديد `budgets`
`id` (uuid) · `categoryId` · `month` (`yyyy-MM`) · `amount` (سنتات) · `currency` (ISO) · `createdAt` · `updatedAt`

- لا يُسمح بأكثر من ميزانية لنفس `(categoryId, month, currency)` — يفرضه المستودع منطقيًا (تحقّق قبل الإضافة).
- الفهارس: `categoryId`, `month`, ومركّب `[month+currency]`.

### ترقية `SCHEMA_VERSION` إلى 2
`db.version(2).stores({ ..., budgets: 'id, categoryId, month, [month+currency]' })`. الجداول الأخرى تبقى كما هي (Dexie يحافظ على البيانات عند الترقية).

## 4. المستودع `budgetsRepo`

- `createBudget(input): Promise<Budget>` — `input = { categoryId, month, amount, currency }`؛ يرمي `DUPLICATE_BUDGET` إن وُجدت ميزانية لنفس المفتاح.
- `updateBudget(id, patch): Promise<void>`
- `deleteBudget(id): Promise<void>` (حذف فعلي — الميزانيات قليلة ولا حاجة لحذف ناعم).
- `listBudgets(month): Promise<Budget[]>`
- `budgetProgress(month): Promise<BudgetProgress[]>` حيث
  `BudgetProgress = { budget: Budget; spent: number; remaining: number; percent: number; status: 'ok'|'near'|'over' }`.
  المصروف يُجمع من العمليات عبر ربط عملة الحساب (كما في `rangeTotalsByCurrency`).

## 5. النسخ الاحتياطي

- `exportBackup` يضيف `budgets` إلى المخرجات (الإصدار 2).
- `importBackup` يقبل `schemaVersion ∈ {1, 2}`؛ عند v1 تُعامَل `budgets` كـ `[]`. يبقى الاستبدال الكامل مع تأكيد.

## 6. الواجهة

- **صفحة الميزانيات `/budgets`:** قائمة ميزانيات الشهر الحالي، كل عنصر يعرض اسم التصنيف، شريط تقدم ملوّن (أخضر/أصفر عند near/أحمر عند over)، «المصروف / القيمة»، المتبقي، والنسبة. أزرار إضافة/تعديل/حذف.
- **`BudgetForm`:** اختيار تصنيف مصروفات، المبلغ، العملة (افتراضي من الإعدادات). يمنع التكرار برسالة عربية.
- **التنقّل:** إضافة تبويب «الميزانيات» إلى الشريط السفلي (يصبح 5 تبويبات).
- **لوحة التحكم:** قسم «تقدم الميزانية» يعرض أهم ميزانيات الشهر (حتى 3) بأشرطة تقدم، مع رابط للصفحة الكاملة.

## 7. الاختبار

- **وحدة (Vitest + fake-indexeddb):** ترقية المخطط واستمرار البيانات، `budgetsRepo` (CRUD، منع التكرار، حساب التقدم عبر العملات وعتبات الحالة)، النسخ الاحتياطي (تضمين الميزانيات، استيراد v1/v2).
- **مكونات:** عرض `BudgetsPage` للميزانيات، ومنع التكرار في النموذج.

## 8. معايير القبول

1. ترقية المخطط إلى v2 دون فقدان بيانات v1.
2. إنشاء/تعديل/حذف ميزانية شهرية لتصنيف، مع منع التكرار لنفس (تصنيف+شهر+عملة).
3. حساب صحيح للمصروف/المتبقي/النسبة من عمليات الشهر، مجمّعًا بعملة الميزانية.
4. حالة بصرية صحيحة عند 80% و100%.
5. لوحة التحكم تعرض تقدم الميزانية.
6. النسخ الاحتياطي يضمّن الميزانيات، والاستيراد يقبل v1 و v2.
