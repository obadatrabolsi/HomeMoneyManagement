# التصدير والاستيراد (CSV/PDF) — مواصفة تصميم (المرحلة الثالثة، الجزء A)

> التاريخ: 2026-06-30
> النطاق: تصدير/استيراد CSV للعمليات + تصدير PDF عبر طباعة المتصفح. يبني على `master` الحالي (بعد إعادة التصميم البصري للجلسة الأخرى).
> لا تغيير على مخطط Dexie أو النسخ الاحتياطي JSON.

## 1. نظرة عامة

تمكين المستخدم من تصدير عملياته إلى CSV (يفتحه Excel مباشرةً)، واستيراد عمليات من CSV، وحفظ التقارير كـ PDF عبر مربّع طباعة المتصفح (يدعم العربية RTL تلقائيًا). كل ذلك client-side ودون اتصال.

## 2. القرارات المعتمدة

| القرار | الاختيار |
| --- | --- |
| الصيغ | CSV (تصدير + استيراد) — يفتحه Excel/Sheets؛ «Excel» = CSV (تجنّبًا لمكتبة xlsx ومخاطرها) |
| PDF | عبر `window.print()` مع نمط طباعة يُخفي شريط التنقّل/الأزرار العائمة — المتصفح يحفظ كـ PDF |
| ترميز CSV | UTF-8 مع BOM لضمان عرض العربية في Excel؛ صيغة RFC 4180 (اقتباس الحقول التي تحوي فاصلة/سطر/اقتباس) |
| أعمدة التصدير | `date,type,amount,currency,account,category,merchant,notes,tags` (المبلغ بالوحدات الكبرى مثل 12.50؛ الوسوم مفصولة بـ `|`) |
| الاستيراد | income/expense فقط؛ تُتجاهل صفوف `transfer` (للحفاظ على ترابط طرفي التحويل)؛ يُطابَق الحساب/التصنيف بالاسم مع الموجود (يُتجاهل الصف إن لم يوجد الحساب)؛ التصنيف اختياري |
| نتيجة الاستيراد | `{ imported, skipped, errors }` تُعرض للمستخدم |
| التخزين | لا جداول جديدة؛ القراءة عبر المستودعات الحالية والكتابة عبر `createTransaction` |

## 3. الوحدات

### `src/lib/csv.ts`
- `toCsv(rows: string[][]): string` — يبني CSV بصيغة RFC 4180 (اقتباس عند وجود `, " \n`؛ مضاعفة الاقتباس).
- `parseCsv(text: string): string[][]` — يحلّل CSV (يدعم الحقول المقتبسة متعددة الأسطر والفواصل المقتبسة)؛ يتجاهل BOM إن وُجد.

### `src/db/dataPortRepo.ts`
- `transactionsToCsv(): Promise<string>` — يبني صفوف التصدير (ترويسة + كل العمليات غير المحذوفة، مع أسماء الحساب/التصنيف) عبر `toCsv`، مع BOM.
- `ImportResult = { imported: number; skipped: number; errors: string[] }`.
- `importTransactionsCsv(text: string): Promise<ImportResult>` — يحلّل، يتحقّق، ويُدرج income/expense عبر `createTransaction`؛ يتجاهل التحويلات والصفوف غير الصالحة (مع رسالة).

## 4. الواجهة

- في صفحة الإعدادات (`BackupPage`) — بطاقة «تصدير/استيراد البيانات» (باستخدام مكونات التصميم الحالية `Card`/`Button` ورموز الألوان): أزرار **تصدير CSV** · **استيراد CSV** (مُنتقي ملف) — مع عرض نتيجة الاستيراد.
- في صفحة **التقارير** (`/reports`): زر **حفظ PDF** يستدعي `window.print()`.
- **نمط طباعة** في `src/index.css` (`@media print`) يُخفي شريط التنقّل السفلي والأزرار العائمة وأي عناصر `.no-print`، ويُظهر المحتوى فقط.
- مفاتيح i18n عربية جديدة حسب الحاجة.

## 5. الاختبار

- **وحدة:** `csv` (toCsv/parseCsv: اقتباس، فواصل/أسطر داخل الحقول، BOM، دورة ذهاب-إياب)؛ `dataPortRepo` (بناء صفوف التصدير، استيراد ناجح، تجاهل التحويلات/الحسابات المفقودة، عدّ النتائج).
- **مكونات:** عرض أزرار التصدير/الاستيراد في الإعدادات (مطابقة لمكونات التصميم).

## 6. معايير القبول

1. تصدير CSV ينزّل ملفًا صالحًا (UTF-8+BOM) يفتحه Excel ويعرض العربية.
2. استيراد CSV يُدرج عمليات income/expense صحيحة، يتجاهل التحويلات والحسابات المفقودة، ويُبلّغ بالعدد.
3. دورة ذهاب-إياب: تصدير ثم استيراد يعيد إنشاء عمليات income/expense.
4. زر «حفظ PDF» يفتح مربّع الطباعة بنمط نظيف (دون شريط التنقّل).
5. لا تغيير على المخطط أو النسخ الاحتياطي JSON.
